# Contributing

If you are planning to contribute to this project or related projects, please contact
the lead maintainer (Matt) on Discord before beginning any work.

You can find the lead maintainer on the [Pirate Software Discord](https://discord.gg/piratesoftware).

Their username is `@mattermatter`. (`<@254671837219389441>`)

Please reach out by direct messaging [with relevant details](https://www.nohello.com/).
(You can find their profile by pinging them by the above username or id)

Friend requests will be ignored. If you are not in the Pirate Software Discord, you may
also be ignored.

If you are having trouble finding them or reaching out, you can open a Moderation Ticket
on the Pirate Software Discord and ask for help.

If you wish to report a bug, please open a moderation ticket on the Pirate Software
Discord with as much detail as possible, including:

- A detailed description of the issue
- Steps to reproduce the issue
- Screenshots or videos demonstrating the issue (can be added after the ticket is open)
- Any relevant logs, error messages, etc.

The moderators can be trusted and will be able to get the information to the relevant
parties quickly.

If you are forking the project for your own use, feel free to ask the lead maintainer
any questions! They cannot guarantee help, but they will do their best to assist you if
they are able.

## Development Strategy

The overall goals for the project are defined by Thor (Pirate Software) who communicates
with the lead maintainer. The lead maintainer is responsible for day-to-day management
and impelmentation details. As such, any updates you wish to implement which are not
already defined in the project roadmap should be discussed with the lead maintainer who
will be able to give feedback on your proposed changes, including runnint them by Thor
if necessary. Please avoid reaching out to Thor directly, since their DMs are usually
full and doing so can complicate communication.

## Ferret Software Wiki

The Ferret Software Extension is fundamentally dependent on the Ferret Software Wiki.
However, the wiki is not managed by the same team as the extension. The lead maintainer
is able to communicate issues with the Pirate Software Wikis team.

If you find an issue with the Ferret Software wiki itself, you can report it via a wiki
ticket on the Pirate Software Discord.

The Wiki team is generally very receptive to feedback and suggested changes which would
benefit the extension, so please do not hesitate to suggest anything you think would
be helpful by talking to the lead maintainer who can communicate with the Wiki team.

## Deployment

When a new version of the extension is ready for deployment, the lead maintainer will
build the release and send it to Thor for review and for him to publish it to Twitch.

Updates to fs-dts are deployed manually by the lead maintainer.

Updates to fs-data are automatically published to npm via Github Actions when the
version number is updated via `npm version` and pushed to the main branch.
